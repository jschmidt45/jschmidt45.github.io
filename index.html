Artifact Selection and Improvement Plan

The first step of this process was to select artifacts that we would improve to showcase our abilities in the three areas described above. I chose to use two straightforward assignments from a previous course.


Task Service

The first assignment I chose was called TaskService. It is a Java project that allows users to create a task object with parameters such as description and name. I selected this artifact to demonstrate my growth and ability regarding data structures and algorithms. The original data structure that held the tasks was a hash map and while that worked well for simply storing the tasks, hash maps do not have the ability to sort their contents. For myself, if I was using a program that allowed me to store and see tasks, I would want the ability to sort and view them based on metrics such as priority and deadline. So, my planned improvement for this artifact was to first update the task object to include date and priority parameters, and then implement tree maps, which would automatically sort them based on these parameters. That way, the program could return tasks sorted by the parameter selected. You can see the original artifact below.

Appointment Service

The second assigment that I chose was called AppointmentService. Like TaskService, AppointmentService was a simple Java program that allowed users to create an appoinment with the parameters of ID, date, and description. My idea for this project was to integrate it with a functional frontend and database to create an entire application with the original AppointmentServive being the basis for my backend code. With this artifact, I planned on demonstrating my growth in both software engineering and design, as well as databases. My planned improvement was to create a frontend using React that had a login page, a signup page, and the overall appointments page. I then planned to implement a MySQL database to store the information for users and appointments. Then, I would integrate Spring Boot to tranform the AppointmentService java file into a RESTful API that would allow my frontend to interact with the database.
